Asset Repair Ticket
This ticket will be created by the IT operator to track record of asset repair/maintenance. This repair has 2 versions depending if repair is done inside and outside the company. Assets can be repaired inside the company and outside. If inside, no need for service fee unless there is an upgrade. If outside, service fee and component cost will be listed.

Category
Option 1: Service only, no cost needed if inside the company just record of repair done and no need for ticket. If outside, service will have a fee and need for a ticket to send cost on bms for approval.

Option 2: If upgrade, component in inventory and component to buy cost will be sent to bms for approval. Also, if new acquirers, bms must provide an order number which is from a financial operator.

Repair Type / Sub-Category
Green sub-category required approval from bms as it has cost and need order number

Choice
Definition
Real-World Example
Corrective Repair
The asset is broken and you are fixing it to restore functionality.
Fixing a loose hinge; soldering a power jack.
Preventive Maintenance
The asset works fine, but you are servicing it to keep it that way.
"PM Schedule": Cleaning dust from server fans; applying new thermal paste.
Upgrade
The asset works, but you are making it better or faster.
Increasing RAM from 8GB to 16GB; installing a GPU.
Part Replacement
A specific component failed and was swapped out.
Dead battery replaced with a new one.
OS Re-imaging
Purely software-level fix for viruses or slow systems.
Reformatting a laptop that has become slow or infected.
Warranty Service
You didn't fix it yourself; you sent it to the vendor.
Sending a MacBook to Apple for a recall issue.


Asset Repair Request Ticket Propose Workflow:
Repair (Outside Map and has Cost) :
IT Support creates ticket for Corrective Repair Ticket
HDTS Ticket Coordinator/Admin checks submitted ticket
If approve, send to TTS for document review
If rejected, closed ticket
TTS Review Document. Agent: Document Controller
If approve, send to bms for approval of budget
If rejected, closed ticket
BMS will review the budget
If approve, return order number
If rejected, closed ticket
Once order number are present, ticket will be pass to AMS
AMS will record the repair 
Set Status of repair “In Progress” and will automatically change its status to “Completed” once the end date. Use formula using start date and end date
Once the repair status changes into “Completed”, the ticket must automatically set its status to resolve and update employees. 
Repair Inside Map :
Same approach to paid but skip BMS part. It include process with component include (Upgrade and Part Replacement)

Asset Incident Request Ticket Propose Workflow:
Employee/HR Employee creates ticket for incident report on asset.
HDTS Ticket Coordinator/Admin will review the submitted ticket.
If reject, close the ticket
If approve, forward to TTS
TTS agent for document controller will review the ticket
If tickets are stolen and damaged by an employee, forward the ticket to bms for employee charge.
If ticket are employee resign or die, forward ticket to HR agent for verification
If BMS and HR agent reject, close the ticket. If approved, forward to AMS.
Once AMS receive the Ticket, IT operator will schedule and audit using schedule checkout date by requestor
Once the asset has been check-in physically, IT operator will perform an audit and update Asset Status.
Once the status of the asset in the incident changes, tickets are now resolved and the ticket must update the employee about the resolved ticket to give feedback.

Asset Disposal Request Ticket Propose Workflow:
AMS Operator / Admin Employee will submit a ticket of asset to be disposed.
HDTS Ticket Coordinator/Admin will verify the submitted tickets.
If approve, forward to TTS
If reject, closed the ticket
TTS will review the submitted documents. Agent: Document Controller.
If Approve, forward to BMS for  Journal Entries
If rejected, closed the ticket
Once BMS receive the ticket for disposal, they will perform journal entries. Once done, include command notes before forwarding to AMS.
Once the AMS receives the ticket, AMS Admin will perform bulk update on status of the following assets into Disposed or Write-off. 
Once updated, AMS admin will click resolve ticket action, update ticket status into resolved.
